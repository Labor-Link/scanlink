.NET Framework

1. 支援的平台

  1. Windows

2. 需求

  1. 需要 Microsoft Visual Studio 2008 或更新的版本, 才能編譯/執行 ARGOX SDK 中的範例程式.
  2. 需要 Windows XP 或更新的版本.
  3. 需要 Microsoft .NET Framework v3.5 Service Pack 1: 安裝 Windows .NET Framework 3.5 Service Pack 1

3. 支援的功能

  1. 支援的模擬器
    • PPLB
    • PPLZ
  2. 支援的傳輸介面
    • 檔案
    • 序列埠
    • 並列埠
    • USB
    • TCP/IP

4. 目錄內容

    /windows_desktop
    • /Doc => 相關的說明文件
    • /Example => 範例程式, 包含 C#
      • /src => 範例程式的程式碼
      • /release => 範例程式的可執行檔
    • /Library => 類別庫和給智能感知功能 (IntelliSense) 用的 XML 檔

5. 提示和建議

  1. 執行立象科技的範例應用程式或是任何其他使用 ARGOX SDK 開發的應用程式, 必須在桌上型系統上安裝 Microsoft .NET Framework v3.5 版本. 如果沒有安裝的話則會出現一個訊息, 告知此應用程式需要一個 Microsoft .NET Framework 版本安裝在桌上型系統上.
  2. 建議將此類別庫放在和應用程式同一目錄, 這樣子可避免應用程式無法使用或找不到此類別庫的問題.
  3. 建議將 XML 檔和類別庫放在一起, 這樣子就會支援智能感知功能 (IntelliSense) 功能, 使你更加方便的使用此類別庫.
  4. 當執行結果和預期不同時, 請利用檔案的傳輸介面, 你將會得到一個原本應該傳送至印表機的指令檔案, 請利用它來確認指令的正確性.
  5. 當你想要使用的傳輸介面不支援時, 請利用檔案的傳輸介面, 你將會得到一個原本應該傳送至印表機的指令檔案, 或是利用函式取得 byte[] 的指令資料, 然後由你來控制傳輸介面, 將檔案內容或取得的 byte[] 指令資料經由你控制的傳輸介面來傳送.

Java

1.1. 支援的平台

  1. Android

1.2. 需求

  1. 需要 Android Studio 3.4.1 或更新的版本, 才能編譯/執行 ARGOX SDK 中的範例程式.
  2. 需要 Android 4.3(API Level 19) 以上的系統版本, 才能執行 ARGOX SDK 中的範例程式.
  3. USB 連線需要手機支援 USB HOST 功能
  4. 藍芽連線需要手機支援藍芽 3.0 以上

1.3. 支援的功能

  1. 支援的模擬器
    • PPLB
    • PPLZ
  2. 支援的傳輸介面
    • 檔案
    • USB
    • TCP/IP
    • 藍芽

1.4. 目錄內容

    /Android_mobile
    • /Javadoc => 相關的說明文件
    • /Example => 範例程式, 包含 Android Studio
      • /src => 範例程式的程式碼
      • /release => 範例程式的安裝檔
    • /Library => aar 檔/jar 檔和給智能感知功能 (IntelliSense) 用的 jar 檔

1.5. 提示和建議

  1. 當執行結果和預期不同時, 請利用檔案的傳輸介面, 你將會得到一個原本應該傳送至印表機的指令檔案, 請利用它來確認指令的正確性.
  2. 當你想要使用的傳輸介面不支援時, 請利用檔案的傳輸介面, 你將會得到一個原本應該傳送至印表機的指令檔案, 或是利用函式取得 byte[] 的指令資料, 然後由你來控制傳輸介面, 將檔案內容或取得的 byte[] 指令資料經由你控制的傳輸介面來傳送.
